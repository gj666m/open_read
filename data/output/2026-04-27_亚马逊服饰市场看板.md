# 需求分析报告：亚马逊女装新品市场看板

> 分析日期：2026-04-27
> 需求来源：需求方（从事亚马逊女装类目）
> 复杂度评级：中等

---

## 1. 需求概述

### 一句话目标
自动采集亚马逊女装类目新品榜单数据，以可视化看板的形式每周呈现市场动态和关键词趋势。

### 需求背景
需求方每周需手动浏览亚马逊 New Releases 榜单和 Instagram 博主分享，了解市场情况和流行趋势，以此辅助判断"接下来可以做什么款"。目前这个过程纯手工、耗时长，希望通过自动化手段至少节省 10% 的时间。本期只做亚马逊部分，Instagram 不在范围内。

### 需求来源
- 原始需求方：亚马逊女装类目运营人员
- 沟通方式：多轮对话确认
- 原始材料：`data/需求信息/亚马逊平台服饰市场了解.md`

### 需求确认过程
经过 8 个问题的澄清，已全部确认：

| # | 问题 | 结论 |
|---|------|------|
| 1 | 亚马逊榜单 | New Releases 新品榜单 |
| 2 | 筛选标准 | 大类排名 3 万之前的产品 |
| 3 | 记录字段 | 款式图、产品链接、开售时间、大类排名、售价、评分、评论数量、FBM/FBA、面料成份、关键词 |
| 4 | Instagram | 本期不做 |
| 5 | 数据产出形式 | 看板 |
| 6 | 更新频率 | 每周一次 |
| 7 | 交付形态 | 可视化看板（网页形式，打开浏览器即可访问） |
| 8 | 关键词分解 | 每个产品拆出多个关键词 |

## 2. 业务规则

### 核心业务流程

```
每周定时触发（如每周一凌晨）
    │
    ▼
① 采集亚马逊女装类目 New Releases 榜单产品列表
    │
    ▼
② 筛选大类排名 < 30,000 的产品
    │
    ▼
③ 逐个访问产品详情页，提取 10 个字段
    │
    ▼
④ 数据清洗（格式统一、去重、关键词分解）
    │
    ▼
⑤ 存入数据库（标记采集周次，保留历史）
    │
    ▼
⑥ 看板自动展示最新数据（新品列表、关键词排行、统计概览）
```

### 业务规则列表

| 编号 | 规则描述 | 条件 | 备注 |
|------|---------|------|------|
| R1 | 只采集 New Releases 榜单 | 女装类目（Women's Clothing） | |
| R2 | 大类排名阈值筛选 | 排名 < 30,000 | 超过的不采集 |
| R3 | 关键词分解 | 每个产品提取多个关键词 | 从产品标题/描述/搜索词中提取 |
| R4 | 每周更新一次 | 固定周期采集 | 历史数据永久保留 |
| R5 | FBM/FBA 识别 | 从产品详情页提取 | 需要访问详情页 |
| R6 | 产品去重 | 以产品链接（ASIN）为唯一标识 | 同一产品不同周次记录排名变化 |
| R7 | 面料成份提取 | 从产品描述/bullet points 中解析 | 可能存在字段缺失的情况 |

### 数据流向

```
Amazon New Releases 榜单页
    → 爬虫脚本（获取产品列表 + 初步排名）
        → 产品详情页（抓取完整字段）
            → 数据清洗（格式化、去重、关键词提取）
                → SQLite 数据库（结构化存储）
                    → Streamlit 看板（可视化展示）
```

### 数据字段定义

| 字段 | 来源 | 类型 | 示例 | 备注 |
|------|------|------|------|------|
| 款式图 | 产品详情页主图 | 图片URL | https://m.media-amazon.com/... | 取第一张主图 |
| 产品链接 | 产品ASIN拼装 | URL | https://www.amazon.com/dp/B0XXXXX | 用 ASIN 作为唯一标识 |
| 开售时间 | 产品详情页 | 日期 | 2026-04-15 | Amazon First Available 字段 |
| 大类排名 | 产品详情页 | 整数 | 25847 | 如 #25,847 in Clothing, Shoes & Jewelry |
| 售价 | 产品详情页 | 浮点数 | 22.99 | 单位 USD |
| 评分 | 产品详情页 | 浮点数 | 4.3 | 满分 5.0 |
| 评论数量 | 产品详情页 | 整数 | 156 | |
| FBM/FBA | 产品详情页 | 枚举 | FBA | 从 Fulfillment 信息判断 |
| 面料成份 | 产品描述/Bullet Points | 文本 | 95% Cotton, 5% Spandex | 可能缺失 |
| 关键词 | 产品标题+描述提取 | 文本列表 | ["yoga pants", "flare leggings"] | 多值字段 |

### 边界条件与异常场景

| 场景 | 处理方式 | 优先级 |
|------|---------|--------|
| 亚马逊反爬/封 IP | 使用请求头伪装 + 请求间隔 + 代理池，失败后重试 3 次 | P0 |
| 产品缺少某个字段（如面料成份未填写） | 记录为 NULL，不跳过该产品 | P1 |
| 某周采集完全失败 | 保留上周数据，标记本周缺失，发送通知 | P0 |
| 同一产品重复出现（跨周） | 按 ASIN 去重，追加排名变化记录 | P1 |
| 榜单为空或结构变化 | 记录异常日志，发送通知，不覆盖旧数据 | P0 |
| 价格显示为区间价（如 $22.99 - $29.99） | 取最低价 | P2 |

## 3. 技术方案

### 技术选型

| 决策点 | 选择 | 理由 | 备选方案 |
|--------|------|------|---------|
| 数据采集 | Python 自建爬虫 | 先验证可行性，成本为零 | 如效果不佳，切换到 Rainforest API / Keepa API |
| 数据存储 | SQLite | 轻量、零配置、数据量适合 | 数据量增大后可迁移到 PostgreSQL |
| 看板前端 | Streamlit | Python 生态、开发快、满足需求 | 后续可升级为 React + API 后端 |
| 定时任务 | 系统级 cron / Python schedule | 简单可靠 | 部署到服务器后使用 cron |
| 编程语言 | Python 3.10+ | 爬虫生态成熟 + Streamlit 原生支持 | |

### 系统对接

| 对接方 | 对接方式 | 关键参数 | 已确认信息 |
|--------|---------|---------|-----------|
| Amazon New Releases 页面 | HTTP 爬虫（requests + BeautifulSoup） | 请求频率控制、User-Agent 伪装 | 目标页面：女装类目 New Releases |
| Amazon 产品详情页 | HTTP 爬虫 | ASIN 拼装 URL | 字段提取逻辑已定义 |

### 关键算法/逻辑

#### 关键词提取逻辑（伪代码）

```
输入：产品标题 + Bullet Points + 产品描述
处理：
  1. 从标题中提取核心词组（去除品牌名、尺寸、颜色等通用词）
  2. 从 Bullet Points 中提取特征词
  3. 去重、统一小写
  4. 过滤掉停用词（如 the, a, and, for, with 等）
  5. 输出关键词列表
输出：["yoga pants", "flare leggings", "high waist"]
```

#### 排名提取逻辑（伪代码）

```
输入：产品详情页 HTML
处理：
  1. 查找 "Best Sellers Rank" 区域
  2. 提取 Clothing, Shoes & Jewelry 大类排名
  3. 解析数字（如 "#25,847" → 25847）
  4. 判断是否 < 30000
输出：排名数值 或 NULL
```

## 4. 任务分解

### Phase 规划

| Phase | 目标 | 涉及模块/文件 | 验证方式 | 依赖 |
|-------|------|-------------|---------|------|
| P1 | 爬虫原型验证 | `scraper/amazon.py` | 能成功抓取 New Releases 页面，提取产品列表和 ASIN | 无 |
| P2 | 详情页数据提取 | `scraper/product.py` | 能从产品详情页提取全部 10 个字段（含关键词分解） | P1 |
| P3 | 数据存储模块 | `database/db.py`, `database/models.py` | 数据正确写入 SQLite，支持按周查询、产品去重 | P2 |
| P4 | 看板展示模块 | `dashboard/app.py`, `dashboard/components/` | 浏览器可访问，展示新品列表、关键词排行、统计概览 | P3 |
| P5 | 定时任务 + 异常处理 | `scheduler/task.py`, `config/` | 每周自动触发采集，失败时有日志和通知 | P1-P3 |

### 预估工作量
- Phase 1（爬虫原型）：验证 Amazon 页面结构，确认采集可行性
- Phase 2（详情页提取）：逐字段调试，重点在关键词提取和面料成份解析
- Phase 3（数据存储）：建表 + 读写逻辑 + 去重
- Phase 4（看板展示）：Streamlit 页面搭建 + 图表 + 筛选
- Phase 5（定时任务）：调度配置 + 异常监控
