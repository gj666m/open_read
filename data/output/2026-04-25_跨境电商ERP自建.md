# 需求分析报告：跨境电商 ERP 自建

> 分析日期：2026-04-25
> 需求来源：公司内部
> 复杂度评级：复杂

---

## 1. 需求概述

### 一句话目标
自建一套跨境电商（亚马逊）ERP 系统，覆盖产品、销售、FBA、采购、仓库、财务、统计、工具、设置等核心模块，替代领星 ERP 以降低成本。

### 需求背景
- 公司目前使用领星 ERP 管理跨境电商业务，费用较高是自建的主要驱动力
- 团队仅 1 人（业务方）+ Claude Code（全额开发），用户负责业务引导
- 不需要功能完全对标领星，但核心框架和基础功能要具备，后续迭代完善
- 先本地跑通，再上云部署

### 需求来源
- 原始需求方：公司内部
- 沟通方式：对话 + 文档
- 原始材料：
  - `data/需求信息/自建erp.md` — 原始需求描述
  - `data/需求原始资料/领星erp参考.md` — 领星 ERP 完整模块结构参考

## 2. 业务规则

### 2.1 模块结构与优先级

基于领星 ERP 参考和业务实际需要，划分为 10 个模块，按优先级分 3 批实施：

#### P0 — 基础框架（必须首先完成）

| 模块 | 核心子功能 | 说明 |
|------|-----------|------|
| **设置** | 店铺授权、用户管理、角色权限、部门管理、仓库设置、汇率管理、操作日志 | 系统基础，所有模块依赖 |
| **产品** | 产品管理（SKU/ASIN）、品牌管理、分类管理、图片管理、属性管理、UPC管理、产品配对 | 产品是所有业务的起点 |
| **销售** | Listing管理、订单列表、售后订单、自发货管理、运营日志 | 核心业务流 |
| **财务** | 利润报表、成本管理、费用管理、往来管理（请款/收款）、结算中心 | 核心业务流 |

#### P1 — 进销存链路（依赖 P0）

| 模块 | 核心子功能 | 说明 |
|------|-----------|------|
| **FBA** | 发货计划、发货单、FBA货件、补货建议、头程分摊、FBA地址簿 | 亚马逊核心物流 |
| **采购** | 采购单、采购计划、供应商管理、退货单、委外订单 | 供应链上游 |
| **仓库** | 即时库存、入库/出库、调拨、调整、收货、质检、FBA库存明细、海外仓 | 供应链中游 |
| **装箱/扩展** | 装箱任务、扫码装箱、成本补录、加工单、盘点、打印条码标签 | 仓库扩展 |

#### P2 — 数据分析与工具（依赖 P0+P1 数据积累）

| 模块 | 核心子功能 | 说明 |
|------|-----------|------|
| **统计** | 进销存报表、销售看板、ASIN360、利润统计、亚马逊源报告、产品表现 | 数据驱动决策 |
| **工具** | 关键词排名、跟卖监控、竞品监控、调价助手、预警规则、审批工作台、AI工具 | 运营效率提升 |

### 2.2 核心业务流程

```
产品管理 → Listing刊登 → 订单产生 → 发货(FBA/自发货)
                                        ↓
供应商管理 → 采购下单 → 仓库收货 → 质检入库 → 装箱 → FBA发货
                                        ↓
                              库存管理 ←→ 补货建议
                                        ↓
                              财务结算 → 利润核算
```

### 2.3 核心数据实体

| 实体 | 关键字段 | 关联 |
|------|---------|------|
| Product | SKU, ASIN, UPC, 品牌ID, 分类ID, 成本, 售价, 图片, 状态 | 产品模块核心 |
| Store | 店铺名, 平台, 站点, 授权状态, API凭证 | 平台授权 |
| Listing | ASIN, SKU, 店铺ID, 标题, 价格, 库存, 状态 | 连接产品与销售 |
| Order | 订单号, 店铺ID, SKU, 数量, 金额, 状态, 日期 | 销售模块核心 |
| PurchaseOrder | 采购单号, 供应商ID, SKU, 数量, 单价, 状态 | 采购模块核心 |
| Inventory | SKU, 仓库ID, 数量, 批次, 在途数量 | 仓库模块核心 |
| Shipment | 发货单号, SKU, 数量, 目标仓(FBA), 物流商, 状态 | FBA模块核心 |
| Supplier | 供应商名, 联系方式, 结算方式, 1688账号 | 采购上游 |
| FinanceRecord | 类型, 关联单号, 金额, 币种, 汇率, 日期 | 财务模块核心 |

### 2.4 系统对接

| 对接方 | 用途 | 对接方式 | 优先级 |
|--------|------|---------|--------|
| Amazon SP-API | 订单同步、库存同步、Listing管理、FBA货件、报告下载 | REST API (OAuth 2.0) | P0 |
| Amazon Ads API | 广告数据、PPC管理 | REST API | P2 |
| 1688 Open API | 采购下单、供应商协同 | REST API | P1 |
| Google Trends | 搜索趋势数据 | 爬虫/第三方 | P2 |

## 3. 技术方案

### 3.1 技术选型

| 决策点 | 选择 | 理由 | 备选方案 |
|--------|------|------|---------|
| 后端框架 | **FastAPI** (Python) | 用户熟悉 Python；FastAPI 性能优秀、自动生成 API 文档、异步支持、类型安全 | Django (更重但 admin 开箱即用) |
| ORM | **SQLAlchemy 2.0** | Python 生态最成熟的 ORM，异步支持好，与 FastAPI 配合完美 | Tortoise ORM |
| 数据库 | **PostgreSQL** | ERP 需要复杂查询和事务，PG 是最佳选择；JSON 支持好 | MySQL |
| 前端框架 | **React + Ant Design Pro** | 企业级后台 UI 组件库，表格/表单/图表开箱即用，国内开发者熟悉 | Vue3 + Element Plus |
| 状态管理 | **Zustand** | 轻量，学习成本低 | Redux Toolkit |
| 构建工具 | **Vite** | 开发体验好，启动快 | Next.js (更重) |
| 缓存 | **Redis** | 会话、缓存、任务队列 | — |
| 任务队列 | **Celery + Redis** | 异步任务（数据同步、报告生成） | arq (更轻量) |
| 部署 | **Docker Compose** | 本地开发一键启动，云部署也方便 | — |
| API 对接 | **python-amazon-sp-api** | Amazon SP-API 的 Python 封装库 | 自行封装 |

## 4. 任务分解

### Phase 规划

| Phase | 目标 | 核心模块 | 验证方式 | 依赖 |
|-------|------|---------|---------|------|
| **P0-0** | 项目初始化：前后端脚手架 + 数据库 + Docker + 认证 | 基础框架 | docker-compose up 能启动，能登录 | 无 |
| **P0-1** | 设置模块：用户/角色/权限/店铺授权 | 设置 | CRUD 用户、授权店铺 | P0-0 |
| **P0-2** | 产品模块：产品 CRUD + 品牌/分类/属性 | 产品 | 产品列表、新建、编辑、搜索 | P0-0 |
| **P0-3** | 销售模块：Listing + 订单同步（Amazon SP-API） | 销售 | 能从 Amazon 拉取订单并展示 | P0-1, P0-2 |
| **P0-4** | 财务模块：利润报表 + 成本管理 + 费用管理 | 财务 | 能根据订单计算利润 | P0-3 |
| **P1-1** | FBA模块：发货计划 + FBA货件 + 补货建议 | FBA | 能创建发货计划、生成货件 | P0-2, P0-3 |
| **P1-2** | 采购模块：采购单 + 供应商管理 | 采购 | 采购 CRUD、供应商管理 | P0-2 |
| **P1-3** | 仓库模块：库存管理 + 入出库 + 调拨 | 仓库 | 实时库存查询、出入库操作 | P1-2 |
| **P1-4** | 装箱扩展：装箱 + 盘点 + 打印标签 | 装箱 | 装箱流程、盘点流程 | P1-3 |
| **P2-1** | 统计模块：报表 + 看板 + 产品表现 | 统计 | 各维度报表展示 | P0 + P1 数据 |
| **P2-2** | 工具模块：监控 + 预警 + AI工具 | 工具 | 关键词监控、预警触发 | P0-0 |

### 各 Phase 验收标准

- **P0-0**: 系统能启动，能注册/登录，JWT 认证正常
- **P0 完整版**: 能管理产品、同步 Amazon 订单、查看基础利润报表
- **P1 完整版**: 能管理采购→仓库→发货的完整链路
- **P2 完整版**: 有数据报表和运营工具
