# 分析报告输出规范

> 用途：定义需求分析报告的精确格式
> 目标读者：另一个项目的 Claude Code 会话（作为项目启动的输入）

---

## 一、设计原则

1. **自包含**：报告本身就是一个完整的项目启动提示词，不依赖其他上下文
2. **机器可读**：结构化到 Claude Code 可以直接解析出技术任务
3. **人可审核**：用户在传给下游项目前能快速审核确认
4. **分级可选**：简单需求用简化版，复杂需求用完整版

---

## 二、文件命名规范

```
data/output/YYYY-MM-DD_需求简称.md

示例：
data/output/2026-04-24_采购对账自动化.md
data/output/2026-04-26_库存预警通知.md
```

---

## 五、复杂度判断标准

| 特征 | 简单 | 中等 | 复杂 |
|------|------|------|------|
| 预估工作量 | 1-2天 | 1-2周 | 1月+ |
| 涉及系统 | 单系统/无对接 | 1-2个外部系统 | 3+系统 |
| 待澄清点 | <=2 且简单 | 3-5 个 | 5+ 个 |
| 影响范围 | 单模块/单脚本 | 多模块 | 跨项目 |
| 报告版本 | 简化版 | 完整版 | 完整版 |
