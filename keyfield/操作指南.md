# 操作指南

> 如何使用这个项目进行需求分析

---

## 快速开始

### 1. 收到新需求时

打开本项目的 Claude Code，直接说：

```
我有一个新需求要分析，[粘贴需求内容]
```

或者需求已经放在文件里了：

```
读 data/需求信息/xxx.md，帮我分析这个需求
```

### 2. Claude 会引导你完成分析

按四阶段自动推进：

```
Phase 1 复述确认 → Claude 用自己的话总结需求，列出模糊点
         ↓
Phase 2 深度澄清 → Claude 对每个模糊点给出多种解读，你选择
         ↓
Phase 3 技术方案 → Claude 输出业务规则、技术选型、任务拆解
         ↓
Phase 4 输出报告 → 生成分析报告，保存到 data/output/
```

简单需求会跳过 Phase 2，直接出方案。

### 3. 拿到报告后去执行

打开目标项目的 Claude Code，复制报告末尾的「项目启动指令」粘贴进去即可。

## 目录用途

| 目录 | 放什么 |
|------|--------|
| data/需求信息/ | 你记录的需求笔记、需求方发来的文字 |
| data/需求原始资料/ | 截图、文档、聊天记录等原始材料 |
| data/output/ | Claude 生成的分析报告（自动生成） |
| data/excel/ | 如有Excel模版等参考文件 |

## 常见场景

### 场景1：微信/钉钉收到需求

```
1. 复制聊天记录，直接粘贴到 Claude Code 对话中
2. （可选）截图保存到 data/需求原始资料/
3. 说：「我有一个新需求要分析，[粘贴内容]」
```

### 场景2：需求方发了文档

```
1. 把文档保存到 data/需求原始资料/
2. 在 Claude Code 中说：「读 data/需求原始资料/xxx.pdf，帮我分析这个需求」
```

### 场景3：开完会后自己整理了要点

```
1. 把要点写到 data/需求信息/xxx.md
2. 在 Claude Code 中说：「读 data/需求信息/xxx.md，帮我分析这个需求」
```

### 场景4：继续上次未完成的分析

```
在 Claude Code 中说：「继续上次的XX需求分析」
Claude 会读取 data/output/ 中的报告继续
```

### 场景5：分析完成，准备开发

```
1. 打开 data/output/ 中的分析报告
2. 复制报告末尾「第6章 项目启动指令」的全部内容
3. 打开目标项目的 Claude Code
4. 粘贴该内容，Claude 会自动建立项目文档体系并开始开发
```

---

## 注意事项

- 这个项目**只做需求分析**，不写业务代码
- 如果分析过程中 Claude 开始写代码，提醒它「这里是分析工作台，请到目标项目中执行」
- 每份分析报告都会自动保存在 data/output/，不会丢失
- 简单需求（1-2天的工作量）流程会简化，不会过度流程化
